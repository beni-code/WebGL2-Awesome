                                          Guillermo Ojea Andrei Despinoiu 2 years ago

                                          Hello, in order to do this you need first to pass the viewport size as a uniform in the fragment shader:
                                          uniform vec2 viewPort;

                                          And in your fragment main loop add the following:
                                          vec2 fragCoord = gl_FragCoord.xy / viewPort * vec2(1.0, -1.0);
                                          intensity *= length(vec3(texture(light.flashlight, fragCoord)));

                                          Pass the viewport size to the shader from your program:
                                          lightingShader.setVec2("viewPort", SCR_WIDTH, SCR_HEIGHT);

                                          Don't forget to load the texture, set the sampler in the Light struct (or somwhere else...):
                                          unsigned int flashlight = loadTexture("textures/batsignal.jpg");
                                          lightingShader.setInt("light.flashlight", 3);

                                          // flashlight texture
                                          glActiveTexture(GL_TEXTURE3);
                                          glBindTexture(GL_TEXTURE_2D, flashlight);

                                          I downloaded a "batsignal png" from google and edited it to remove the transparency, invert the colors and blur it.

                                          If you got this far in the tutorials these steps should be enough.

                                                          Andrei Despinoiu ticov 2 years ago edited

                                                          In the above screenshots the "cookie" texture is square, not oval. It's acting as a mask, in screen space. The final color is multiplied by the values in the texture so that it "carves" into the spotlight disc, giving it whatever shape you want, even a Bat signal. This article explains it better: https://en.wikibooks.org/wi...

                                                          Bookmark it because it will become VERY useful when you reach the Shadow Mapping tutorial later on and you have a projection matrix for the spotlight, since the same matrix used for rendering to the depth map can be used for the light cookie, giving you the ability to place them anywhere in the scene, not just as a screen space effect. You can also attach it at the end of an actual flashlight model/weapon and have it "wobble" around as your character moves, or as the headlights of a car, ceiling lamps, etc.

                                                                Andrei Despinoiu Saurabh Bhurewar 2 years ago edited

                                                                The sampler2D didn't use the texture coordinates of the models (cubes in this tutorial) that are being rendered, because then, yeah, it would have been applied to each face, since they share the same texture coordinates. It used a screen-space projection matrix to "align" the cookie texture with the camera, ending up as an overlay (from the camera's point of view) on top of whatever was being rendered. In the fragment shader it used textureProj() instead of texture() (mainly so that I wouldn't need to divide the texture's .xyz by its .w component to get it to sample from the correct place in the texture), and using that matrix as coordinates, only getting the ".r" channel from it, although color could have also been used (except not with a float, obviously, with a vec3 instead).

                                                                float intensity = textureProj(light.flashlightCookie, ProjTexCoord).r * uFlashlightBrightness; // "uFlashlightBrightness" is a float uniform
                                                                diffuse  *= intensity;
                                                                specular *= intensity;

                                                                I'm sorry if I couldn't explain it any better... You need a fair amount of knowledge of how matrices work and linear algebra (I recommend khanacademy.org). I mostly improvised, to be perfectly honest :) and lots and lots of experimenting. Don't worry if you don't get it right now, it will eventually "click", trust me. And there might be other ways than using a projection matrix. Guillermo posted a possible solution above that seems like it could work. Maybe give it a shot. And now that I think about it, it could also be applied as a post-processing effect when you reach the Framebuffers tutorial. There are multiple ways this could be implemented.

                                                                                            Олег Юрченко wang dingqiao 5 years ago

                                                                                            Intensity is the ratio of the cosine of the angle of the fragment to the cosine of the inner angle of the cone. This relationship is based on a certain base value (the cosine of the angle of the outer cone in this case) and always less than the inner angle. If it is given that the outer cosine is zero, the formula becomes I = theta / inner, if theta < inner, then the result is 0..1 (partially shaded), if theta >= inner, then the result will be clamped to 1 (fully lit). But when we have an outer cosine, we have a "new zero", a "new base value", and then we need to calculate theta and inner with respect to the new base value: theta - external and internal - external.
