                                                    You can just update lightPos in the rendering loop (adjust values accordingly to suit your needs)

                                                    // set light position
                                                    float lightX = 2.0f * sin(glfwGetTime());
                                                    float lightY = -0.3f;
                                                    float lightZ = 1.5f * cos(glfwGetTime());
                                                    glm::vec3 lightPos = glm::vec3(lightX, lightY, lightZ);

                                                    // send light position to lightingShader
                                                    lightingShader.use();
                                                    lightingShader.setVec3("lightPos", lightPos);

                                                    // send model/light position to lamp shader
                                                    lampShader.use();

                                                    model = glm::translate(model, lightPos);
                                                    model = glm::scale(model, glm::vec3(0.2f));
                                                    lampShader.setMat4("model", model);

                                                            Ben ChristiePPPPP 3 years ago

                                                            To replicate the lamp orbiting the cube you use the exact same code as in the solution to the first exercise except you change what you update the lightPos in the render loop to.

                                                            // this creates a rotation around the world origin on the xz-plane with a radius of 2
                                                            lightPos.x = sin(glfwGetTime()) * 2.0f;
                                                            lightPos.z = cos(glfwGetTime()) * 2.0f;
                                                            lightPos.y = 0;

                                                            If you are struggling to understand why these equations create the rotation they do I recommend setting one component as above and the rest to 0 so you can see what each is doing. If you still can not determine what is going on you should probably head over to https://www.khanacademy.org and review some math topics. NOTE: flipping sin and cos makes it revolve in the opposite direction. and if you use either sin or cos for both the x and z component then the cube will traverse the line x=z. (A line of slope 1 on the xz-plane)

                                                                                                                    JoeyDeVries Sensei Hirosam 3 years ago

                                                                                                                    That makes no sense to me, modifying the LightDir like that. I'm guessing it's a coincidence it looks 'somewhat alright' for you.

                                                                                                                    Few things to look out for:
                                                                                                                    - When doing lighting in view-space all relevant vectors need to be in view space. Since you're doing diffuse and specular this means light.direction should be in view space, but also the normal vector, and the view vector (doing a reflect on two view-space vectors will already give the result in view-space).
                                                                                                                    - When multiplying world-space vectors with the view matrix you want to make sure whether you're dealing with a direction or a position vector. For a position vector you want its w component to be 1.0 so translation is included. For a direction vector you want the w component to be 0.0. The light, view and normal direction are all direction vectors so make sure to multiply them with a w component of 0.0: like 

                                                                                                                    vec3 lightDir = vec3(View * vec4(light.direction, 0.0))


                                                                                                                    - Also make sure all vectors are unit vectors (normalized) when comparing/dot-producting them.

                                                                                                                    Hope it helps!
