                                                                                                                      Andrei Despinoiu 3 years ago edited

                                                                                                                      To match the camera position like in the screenshot, set this in your render loop and fly around until it matches:
                                                                                                                      std::cout << cameraPos.x << ", " << cameraPos.y << ", " << cameraPos.z << std::endl;
                                                                                                                      Jot down the coordinates in Notepad (or whatever), then do the same thing for the front vector (i.e. where the camera is pointing):
                                                                                                                      std::cout << camera.cameraFront.x << ", " << camera.cameraFront.y << ", " << camera.cameraFront.z << std::endl;

                                                                                                                      In my case they were:
                                                                                                                      cameraPos = glm::vec3(1.16f, 1.44f, 4.41f);
                                                                                                                      camera.cameraFront = glm::vec3(-0.21f, -0.28f, -0.94f); // This line adds a bit of a jump when moving the camera for the first time
                                                                                                                      I added them right before the render loop so I can still fly around the scene, because adding them to the render loop would've made the camera static.

                                                                                                                      Alternatively, you can also set the position when you instantiate it and let the constructor initialize it with those values.
                                                                                                                      Camera camera(glm::vec3(1.16f, 1.44f, 4.41f));

                                                                                                                                                        JoeyDeVries Sensei Chris 6 years ago

                                                                                                                                                        Yes, it's prefectly safe to unbind the VBO after the calls to glVertexAttribPointer. However, the EBO binds are stored in the VAO, so you don't want to unbind the EBO while the VAO is still bound. (after unbinding the VAO you're free to unbind the EBO as well). OpenGL is weird like that.
